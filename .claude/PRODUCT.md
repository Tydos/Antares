# Product

## Register

product

## Users

Recruiters and engineers reviewing a portfolio project. Engineers scan for technical depth: the DevPanel, search mode controls, hybrid RRF search, and latency numbers are the proof points. Recruiters read the overall polish — does this look like something a senior engineer shipped, or a tutorial clone? Both audiences are time-pressured and skeptical. First impressions carry disproportionate weight.

## Product Purpose

A RAG (Retrieval Augmented Generation) PDF chatbot that demonstrates full-stack engineering — vector search, hybrid retrieval, persistent conversation history, and an evaluation pipeline — packaged as a clean, usable product. Success means an engineer opens it and thinks "I'd actually use this" and a recruiter thinks "this person has taste."

## Brand Personality

Confident, minimal, purposeful. The execution is serious. No decorative chrome, no performed sophistication. The UI does exactly what it says and looks good doing it.

## Anti-references

- Generic SaaS purple soup — bright violet everywhere, rounded cards, gradient CTAs
- Dark hacker-terminal aesthetic — neon on black, monospace everything, trying to look clever
- Over-produced landing page kitsch — particle backgrounds, hero animations, scroll reveals for their own sake
- Tutorial-clone flatness — default white + blue links + zero spacing rhythm

## Design Principles

1. **Show the work, don't announce it.** Technical depth (DevPanel, search modes, chunk inspector, latency bars) should be accessible and readable — not buried or flaunted. Let the feature speak.
2. **Restraint signals competence.** An empty state that breathes, a send button that does one thing, a sidebar with no wasted space — these communicate more than decoration.
3. **Every element earns its place.** No filler, no placeholder sections. If it's in the UI, it has a job.
4. **The joke is in the name, not the design.** Irreverence lives in the branding. The interface itself is precise and considered.
5. **Portfolio-grade finish.** Hover states, transitions, error handling, empty states — all shipped, all polished. Nothing half-done.

## Accessibility & Inclusion

WCAG AA minimum. Respect `prefers-reduced-motion` for any animations. Sufficient contrast on all text. Keyboard navigation on all interactive elements.
